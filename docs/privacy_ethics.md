# Privacy and Ethics-by-Design

## Principles

- Use anonymized runtime IDs only; no persistent identity linking.
- Keep processing local by default; avoid cloud upload of raw classroom video.
- Do not store raw face crops unless explicitly approved and documented.
- Treat analytics as pedagogical support, not disciplinary surveillance.

## Default Technical Safeguards

- Reports store aggregate and pseudonymous per-track statistics.
- Realtime stream publishes student IDs as transient track IDs only.
- Run manifests capture reproducibility metadata, not sensitive personal fields.

## Deployment Checklist

- Collect informed consent from institution and participants.
- Publish purpose, retention policy, and intervention policy.
- Set explicit session retention windows for outputs.
- Run bias audits across demographics and camera setups.
- Ensure teacher dashboard wording avoids stigmatizing labels.

## Scope Boundaries

- The system estimates attention-related signals; it does not infer intent or diagnose conditions.
- Outputs should be interpreted with uncertainty and used with human oversight.
